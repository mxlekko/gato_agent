# `_副本` 常驻启动说明

本文档用于固定 `_副本` 的真实端口、重启方式、控制台代理和路径边界，避免再次把请求打到旧仓或旧进程。

## 默认端口组

当前 `_副本` 默认端口由 [`.env`](/Users/gato-pm/Desktop/API_副本/.env) 控制：

- API: `3100`
- ContextHelper: `19101`
- DirectDbRunner: `19102`
- ModelTool: `19103`
- Console dev: `3200`

说明：

- `3100/19101/19102/19103` 是 `_副本` 自己的后端端口
- `3100` 是建议开放给同一内网其他机器访问的统一入口
- `19101/19102/19103` 必须保持 `127.0.0.1` 回环监听，只供本机 API 进程调用
- `3200` 是前端 dev server
- 控制台 `/api/*` 默认代理到 `_副本` API `http://127.0.0.1:3100`
- 如果旧 API 需要并行运行，建议单独放在 `3001`

## 常驻重启

在仓库根目录执行：

```bash
cd /Users/gato-pm/Desktop/API_副本
npm run service:restart
npm run service:status
```

`service:status` 应看到四个 launchd 服务的：

- `working directory = /Users/gato-pm/Desktop/API_副本`
- API 监听 `0.0.0.0:3100`
- ContextHelper 监听 `127.0.0.1:19101`
- DirectDbRunner 监听 `127.0.0.1:19102`
- ModelTool 监听 `127.0.0.1:19103`

## 控制台代理

启动控制台：

```bash
cd /Users/gato-pm/Desktop/API_副本/console
npm run dev
```

默认代理规则：

- `http://127.0.0.1:3200/api/*` -> `http://127.0.0.1:3100/api/*`

如果需要切换到其他实验 API：

1. 参考 [console/.env.local.example](/Users/gato-pm/Desktop/API_副本/console/.env.local.example)
2. 在 `console/.env.local` 中设置 `VITE_API_PROXY_TARGET`
3. 重启 `npm run dev`

## 路径规则

文档、配置、运行时都应遵守同一套路径边界：

- 允许 `project://...`
- 允许 `runtime://project-runtime/...`
- 禁止 `/Users/gato-pm/Desktop/API/...`
- 禁止 `/Users/gato-pm/.openclaw/...`
- 禁止继续新增或使用历史 `runtime://openclaw/...`

其中：

- `project://` 解析到 `_副本` 仓库目录
- `runtime://project-runtime/...` 解析到 [runtime-assets/project-runtime](/Users/gato-pm/Desktop/API_副本/runtime-assets/project-runtime)

如果 scene 配置或 direct-model 资产继续引用旧路径、共享 `.openclaw` 路径或历史 `runtime://openclaw/...` namespace，运行时现在会直接报错。

## 允许继续存在的外部依赖

以下依赖允许保留，但必须显式来自 [`.env`](/Users/gato-pm/Desktop/API_副本/.env) 或受控配置：

- SQL Server
- LLM Provider 凭证
- RAG 服务和本地工具服务

这些不属于 `_副本` 需要复制的本地文件资产。

## 重启后检查

最小检查：

```bash
curl -sS http://127.0.0.1:3100/health
curl -sS http://127.0.0.1:3100/api/console/configs/catalog
```

如果要做完整自闭环检查：

```bash
cd /Users/gato-pm/Desktop/API_副本
npm run regression:self-contained
```

当前该回归的预期是：

- 共享路径扫描 `findings = 0`
- `payment-info-split` pass
- `sales-opportunity-advisor` pass
- `sales-opportunity-advisor-directdb` pass
- `sales-opportunity-smart-entry` pass

销售机会主场景现在走项目内 LangGraph/runtime，不再要求本机 legacy agent gateway。

## 与旧 API 并行

推荐的并行验证方式：

```bash
curl -sS http://127.0.0.1:3001/api/console/configs/catalog
curl -sS http://127.0.0.1:3100/api/console/configs/catalog
```

这样可以同时看到旧 API 和 `_副本` API，但 3200 控制台默认只会打到 `_副本` 的 3100。
